Informações gerais

A partir da versão DSpace 1.2, muitas ferramentas de administração podem agora ser acessadas pelo DSpace padrão UI (User Interface). No entanto, elas só aparecem se você estiver logado como Administrador. A maneira mais fácil de fazer isso é ir para a página "Meu espaço".

Algumas tarefas devem ser realizadas internamente usando a interface do usuário Administrador, parte da Interface do Usuário Web. Para acessar a interface do usuário Administrador, digite a URL base do seu DSpace seguido por 

/dspace-admin
, por exemplo:

https://dspace.myu.edu/dspace-admin

Você vai precisar da autenticação como administrador para ser autorizado a acessar a página. Em seguida, clique na ferramenta respectiva na barra de navegação do lado esquerdo.

Criação de uma Comunidade de Nível Superior

Para adicionar uma comunidade de nível superior, vá para a página da listagem de Comunidade / Coleção quando estiver conectado como administrador. À direita, haverá uma opção Criar Comunidade de Primeiro Nível .... Ao clicar sobre esta opção você criará uma nova Comunidade de nível superior e será levado para a página 'Editar Comunidade'.

Criando uma Sub-comunidade

Para criar uma sub-comunidade, vá para a página inicial da Comunidade pai da nova sub-comunidade. Em seguida, clique no botão "Criar sub-comunidade" na caixa "Ferramentas Admin" no canto superior direito da página. Isto irá criar a nova comunidade e levará para a página 'Editar Comunidade'.

Editando uma Comunidade

  • Preencha o formulário conforme necessário. Os campos 'Nome', 'Descrição curta' e 'Texto introdutório' são obrigatórios; os demais campos são opcionais.
  • A 'Descrição curta' aparece na página da comunidade abaixo do nome da comunidade, e deve ser uma ou duas frases de texto simples que descrevem a comunidade.
  • Os campos "Texto introdutório 'Texto da barra lateral' e 'Texto de copyright' são exibidos na página inicial da comunidade. 'Texto introdutório' e 'Texto da barra lateral' "são os dois campos de HTML, o que significa que você deve colocar o texto em <P> </P>
  • Para adicionar um logotipo a ser exibido na página inicial da comunidade, clique no botão Upload logo. Na próxima tela, digite um caminho para o arquivo do logotipo ou navegue até o arquivo de logotipo, em seguida, clique no botão Upload logo. Note que, se você está criando uma comunidade, é melhor preencher o restante do formulário, clique em "Update" e depois voltar para fazer isso. Note-se que, em geral, os logotipos pequenos são mais adequados devido ao design da home page da coleção.
  • Quando todas as informações para a página da comunidade estiverem inseridas, clique no botão Criar button.

Criando uma coleção

A criação de uma coleção é feita com uma ferramenta de estilo 'wizard' que deve ser familiar para a maioria das pessoas que já usaram interfaces de usuário de sistemas operacionais modernos. Ela é bastante semelhante à interface de submissão no DSpace.

Nota: Você não pode adicionar novos registros de usuários com este assistente. Todos os registros de usuários devem ser carregados primeiro no sistema. No entanto, depois você poderá adicionar mais usuários e / ou grupos a cada grupo associado à coleção, se necessário.

Para criar uma coleção, assegure-se que você está logado, vá para a página inicial da comunidade na qual coleção deve aparecer, e clique em "Criar Coleção” 'no canto superior direito da página. Isto irá iniciar o assistente.

Descrever a Coleção (caixas de seleção)

A primeira página do assistente vai lhe fazer algumas perguntas sobre a nova coleção. Confira as que se aplicam e clique no botão 'Próximo>'. Observe que depois de ter concluído o assistente, você poderá editar a coleção mais tarde, então as decisões que você fizer aqui não são definitivas.

No entanto, observe também que o assistente é um pouco menos sofisticado do que a submissão pela Interface do Usuário, o que significa que não há botões "Voltar", se você quiser mudar algo depois de clicar em 'próximo'. Para isto você precisa concluir a tarefa no assistente e, em seguida, voltar e editar a coleção.

Quando tiver concluído o assistente, você entrará na tela Editar Coleção. Isso permite que você reveja o que digitou e faça os ajustes necessários.

Nota 1: Se a opção Novos itens podem ser legíveis publicamente for marcada, significa que os novos itens submetidos pela interface do usuário ou por importação em lote, terão por padrão, a permissão READ dos usuários anônimos, tanto para o item e todos os bitstreams. Você pode mudar este padrão, para que, por exemplo, os itens possam ter permissões de leitura por usuários anônimos, mas os bitstreams não. Os itens que são "mapeados" ou incluídos partir de outras coleções, não terão suas autorizações alteradas.

Nota 2: Novos itens podem ser legíveis publicamente é apenas um padrão; permissões de um item poderão ser modificadas.

Nota 3: Se a opção Novos itens podem ser legíveis publicamente não é marcada, você será solicitado, posteriormente, a informar no assistente quem tem permissão para ler os novos itens. Novamente, este é apenas um padrão, e pode ser alterado posteriormente.

Descrever a Coleção (Informação)

A próxima página permite que você preencha algumas informações básicas sobre a coleção.

O Nome, a Descrição curta e o Texto introdutório não são campos obrigatórios. Note que, no entanto, que é altamente recomendado o preechimento destes.

Texto de Copyright é um texto que será exibido na parte inferior da home page da coleção.

Licença é a licença de depósito (licença que o depositante deverá conceder quando submete o item) para esta coleção. Esta substitui a licença padrão da Biblioteca Digital (que é armazenada em 

[dspace]/config/default.license
e deve ser editada pelo administrador do sistema). Se você deixar este campo em branco, a licença padrão da Biblioteca Digital será usada.

Procedência é um campo de texto livre no qual você pode digitar qualquer informação de proveniência. Esta informação não é visível para os usuários finais.

Autorização de Coleção

Dependendo das caixas que você marcou na primeira página do assistente, você vai passar por uma série de telas perguntando quais usuários e/ou grupos podem ou devem realizar ações como, por exemplo, quem pode submeter, e quem deve aceitar/rejeitar a submissão no fluxo de trabalho. Essas páginas funcionam exatamente como o editor de grup. Clique em 'Próximo >' uando você tiver selecionado todos os usuários e/ou grupos que aparecem no respectivo grupo de permissões.

Nota: Você não pode adicionar novos registros de usuários com este assistente. Todos os registros de usuários devem ser primeiramente carregados no sistema. Você pode adicionar mais usuários e/ou grupos a cada grupo mais tarde, se necessário.

Digite Metadados Padrão do Item

Nesta tela você pode especificar alguns valores Dublin Core que terão que ser pré-preenchidos para as novas submissões. Alguns dos valores que você definir aqui não aparecem para submissão na interface do usuário, por isso os usuários finais podem não ter a oportunidade de editá-los. Esta pode ser uma coisa boa ou ruim! (FIXME?) Observe também que esses valores Dublin Core serão adicionados aos itens importados por lote.

Editando uma coleção

A página Editar Coleção permite que você altere os metadados da coleção (como Nome e Texto introdutório), configurar a autorização e fluxo de trabalho, ou excluir a coleção inteiramente.

O botão "Apagar' (localizado no topo da página) irá remover a coleção (incluindo todos os seus itens) da Biblioteca Digital.

O Nome, a Descrição curta e o Texto introdutório não são campos obrigatórios. que, no entanto, que é altamente recomendado o preechimento destes.

O Copyright Text is simply text that will appear at the bottom of the collection's home page.

Licença é a licença de depósito (licença que o depositante deverá conceder quando submete o item) para esta coleção. Esta substitui a licença padrão da Biblioteca Digital (que é armazenada em 

[dspace]/config/default.license
e deve ser editada pelo administrador do sistema). Se você deixar este campo em branco, a licença padrão da Biblioteca Digital será usada. Existem algumas variáveis ​​de substituição, que podem ser utilizadas para criar licenças personalizadas:

  • %1$s será traduzido para o "primeiro nome do depositante"
  • %2$s será traduzido para o "último sobrenome do depositante"
  • %3$s será traduzido no "email do depositante"
  • %4$s será traduzido para a data atual
  • %5$s será traduzido na informação da coleção
  • %6$s será traduzido na informação do item
  • %7$s será traduzido na informação do usuário

Se você precisa incorporar no texto da licença o símbolo %, você pode evitar a inclusão de um segundo %, escrevendo %%. Recomenda-se usar uma submissão fictícia para verificar a mudança na licença: verifique a licença resultante na etapa Aceitar / Rejeitar licença.

Procedência é um campo de texto livre você pode colocar qualquer informação de proveniência que lhe convier. Esta informação não é visível para os usuários finais.

Para adicionar ou alterar o logotipo a ser exibido na home page da coleção, clique no botão “Carregar Logo”. Na próxima tela, digite um caminho para o arquivo do logotipo ou navegue até o arquivo do logotipo, em seguida, clique no botão “Carregar”.

Para adicionar ou alterar as autorizações, os depositantes ou administradores de coleção, clicar nos botões respectivos para criar/editar grupos e suas ações.

Para editar os fluxos de trabalho de grupos, selecione a etapa do fluxo de trabalho para o grupo que deseja criar ou alterar e siga o processo editar grupo process

Clique no botão 'Template do Item' para exibir a página de metadados padrão do item onde você pode definir os valores padrão para os campos de metadados para todos os itens adicionados à coleção.

e-Usuário

Utilize esta ferramenta para registrar os endereços de e-mail e outras informações sobre as pessoas que você deseja autorizar a desempenhar um papel no sistema, seja como depositante, administradores, os participantes do fluxo de trabalho de submissão, assinantes ou usuários com permissões para ler um conteúdo restrito.

Os e-usuários também podem ir para esta lista por auto-registro (se a configuração do site permitir esta função). Isso, no entanto, não lhes dá qualquer autorização. Todas as pessoas que você deseja adicionar a um grupo de depositantes ou conceder autorização de workflow devem ser, primeiramente, registradas aqui.

Registre um e-Usuário

Para inscrever alguém, clique no botão Add ePerson e adicionar suas informações.

  • Se o usuário fizer login com uma senha, não use Can Log In, pois isto informa ao sistema que o usuário tem uma conta registrada e ativa, o que não é o caso de um novo e-usuário – que necessitará usar a interface do usuário para registrar uma senha pela primeira vez.

  • A caixa de seleção "exigir certificado' é usada pelo certificado X509 do autenticador MIT. Ignore essa opção, a menos que esteja usando o autenticador MITAuthenticator.

    Se você marcar "exigir certificado', você também pode marcar "pode logar". Isso significa que o usuário pode fazer logon diretamente com o seu certificado, sem registrar. Se você quiser que os usuários passem pelo processo de registro (para que digitem o nome etc), então não marque 'pode logar'.

Clique em Salvar edições quando terminar..

Excluir uma informação E-Usuário ou Editar Informação sobre um E-Usuário

Clique no botão Select EPeople para obter uma lista pop-up de e-usuários no sistema. Os links na parte superior e inferior do pop-up permite que você percorra a lista. Você também pode ordenar a lista por endereço de e-mail, sobrenome ou ID, clicando no cabeçalho da coluna correspondente. Na lista, à esquerda de cada e-usuário há um botão 'selecionar'. Clique próximo do nome do e-usuário correspondente e o pop-up será fechado e e-usuário vai aparecer na caixa da página principal do 'Administer EPeople'. Agora você pode clicar nos botões “deletar” ou "editar" que lhe permitem editar informações sobre um usuário ou para excluí-las do sistema.

Grupos

Use esta seção para criar, editar e excluir grupos de e-usuários e/ou grupos que podem ser autorizados para executar funções específicas no sistema.

O grupo 'Anonymous' é um grupo "especial" que representa cada pessoa que usa o sistema. Este grupo existe para que as políticas de autorização possam ser especificadas para acesso anônimo ("todo mundo pode ler este item).

Os membros do grupo "Administrador" estão autorizados a executar qualquer ação no sistema, por isso pense bem sobre quem serão os usuários que você vai incluir neste grupo!

Outros grupos podem ser usados ​​para uma variedade de fins. Os grupos podem representar e-usuários que têm permissão para submeter a uma coleção particular; e-usuários responsáveis ​​pela realização de uma etapa específica do fluxo de trabalho, como rever as submissões recebidas; ou grupos de e-usuários que têm permissão para acessar algum conteúdo restrito. Informações sobre o que cada grupo tem permissão para fazer são controladas usando a seção de autorização.

Os grupos podem conter outros grupos para proporcionar maior flexibilidade na atribuição de ações e para simplificar a manutenção. Isso é útil para refletir estruturas de grupos hierárquicos, onde um grupo de e-usuários com responsabilidades particulares também faz parte de um grupo maior com diferentes reponsabilidades e/ou escopo de responsabilidade.

Os grupos podem ter nomes. Os nomes criados automaticamente pelo sistema seguem uma convenção de acordo com o objetivo do grupo. A forma geral é:

OBJECTTYPE_OBJECTID_ACTION

Por exemplo, o grupo de e-usuários autorizado a submeter para a coleção XYZ seria denominado:

COLLECTION_XYZ_ADD

Note que XYZ neste caso é o ID interno (a chave primária do banco de dados) da coleção, ao invés do Handle. Você pode descobrir o ID interno de uma coleção ou da comunidade clicando em “Comunidades/coleções”. O segundo número entre parênteses ao lado do nome da comunidade ou coleção é ID interno desse objeto.

É recomendado que você siga essa convenção para grupos criados manualmente, visando facilitar a localização do grupo que você precisa. A interface do administrador criará automaticamente os grupos para as etapas de fluxo de trabalho, mas você precisa criar manualmente os grupos que submeteu.

Criar e editar grupos

Para criar manualmente um grupo, clique em Criar Novo Grupo. Na parte superior altere o nome padrão para o nome desejado e clique em Atualizar o nome.

Ordens de Supervisão

Selecione os supervisores na área de administração e serão apresentadas três opções:

  • Adicionar uma Ordem de Supervisão
  • Ver Ordens de Supervisão atuais
  • Limpar banco de dados de Ordem de Supervisão

As seções a seguir detalham como lidar com tarefas de ordem de supervisão comuns.

Nota: A opção limpar banco de dados de Ordem de Supervisão deve ser usado de vez em quando apenas para executar operações básicas de manutenção no banco de dados. O processo é completamente automático e leva pouco tempo para ser executado.

Vendo Ordens de supervisão atuais

Selecione Exibir Ordens de Supervisão Atuais e será apresentada uma lista de todas as ordens de supervisão em vigor. A partir desta página você pode modificar as políticas para a ordem de supervisão ou remover a ordem de supervisão. (Veja as seções abaixo).

Os detalhes que você tem nesta página permitem que você veja o nome do grupo que faz a supervisão, o autor do item que está sendo supervisionado e o título desse item.

Você também pode adicionar uma ordem de supervisão desta página conforme a seção Adicionando Ordens Supervisão abaixo.

Adicionando Ordens de Supervisão

Selecione Adicionar uma Ordem de Supervisão a partir de qualquer página da ordem de supervisão principal ou da página Ordens de Supervisão atual.

Para definir uma ordem de supervisão você precisa para definir o grupo que fará a supervisão e o item na área de trabalho que será supervisionado. Além disso, esta página lhe dá a opção de definir um conjunto de políticas padrão que o grupo de controle terá sobre o item.

Selecione o grupo desejado da lista pull down nesta página; esta caixa contém todos os grupos que existem atualmente no sistema. Em seguida, selecione a área de trabalho do item que será supervisionado. Isso significa clicar na caixa de seleção redonda à direita do item na listagem da página, e que mostra todos os itens atualmente na área de trabalho do usuário.

Nota: o ID que está na coluna do lado esquerdo do item listado é o ID do banco de dados para a área de trabalho do item, e está lá para a referência dos administradores do sistema.

Uma vez escolhido o grupo e a área de trabalho do item a ser conectado, você também pode selecionar uma política padrão da caixa Initial Policy Setting. As opções de política padrão são:

  • Nenhum - A ordem de supervisão é criada, mas ao grupo de controle não é dada qualquer política em relação ao item a ser supervisionado. Isso geralmente indica que o administrador decidiu criar, manualmente, as políticas de supervisão para este fim.
  • Observador -o grupo de supervisão é dado acesso de READ ao item (mas não a quaisquer pacotes ou bitstreams que já existem). Todos os novos pacotes ou bitstreams herdam a política do grupo de supervisão para permitir operações de leitura.
  • Editor - Ao grupo de supervisão é dado acesso de ADD, WRITE E READ ao item (mas não todos os pacotes ou bitstreams que já existem). Todos os novos pacotes ou bitstreams herdam a política do grupo de supervisão para permitir as operações ADD, WRITE E READ.

Uma vez concluída a seleção, clique em Enviar Ordem de Supervisão e suas configurações serão aplicadas, e você retornará à página principal da ordem de supervisão.

Removendo Ordens de Supervisão

Ir para a página Ver Ordens de Supervisão Atual e selecione Remover da ordem de supervisão a ordem que deseja remover. Você será solicitado a confirmar a remoção da ordem, após a confirmação ela será removida do sistema.

Personalizando Políticas de Ordens de Supervisão

Ir para a página Ver Ordens de Supervisão Atual e selecione Políticas para as ordens de supervisão para o qual você gostaria de ter políticas personalizadas. Você será levado para a página Políticas do item. A partir daí você pode personalizar as políticas para o item específico.

A fim de definir políticas para o grupo supervisor você deve selecionar, por exemplo, Adicionar Nova Política para o item ou qualquer um dos seus componentes, em seguida, selecione da lista de grupos, o grupo supervisor que você quer configurar. Selecione a ação que deseja realizar e clique em Salvar Política.

Registro de Metadados

Por padrão o DSpace usa uma versão qualificada do esquema Dublin Core baseado no Dublin Core Libraries Working Group Application Profile (LAP). Você também pode configurar outros esquemas de metadados no registro. O registro de metadados fornece uma lista dos elementos e qualificadores, com comentários.

Com esta ferramenta você pode editar o registro para atender às suas necessidades, mas note que:

  • Você só pode editar um elemento/qualificador de cada vez. Coloque estas edições em fila, e clique no botão 'Update'. Só edições feitas para essa linha serão salvas.
  • Se você excluir um elemento qualificador, todas as ocorrências desse elemento e de qualificador nos metadados dos itens serão excluídos.
  • Se você atualizar um elemento ou qualificador, por exemplo, mudando de 'contribuidor' para 'criador' e clicando em 'Update', todas as ocorrências de 'contribuidor' nos metadados dos itens serão alterados para "criador".
  • Alguns elementos e qualificadores Dublin Core são utilizados para várias funções pelo sistema, tais como date.available para a coleta OAI, e title para a exibição de item e índice de busca. Você pode alterar as notas de escopo para estes, mas você não deve alterar ou remover o elemento ou qualificador. Para ver quais elementos / qualificadores são utilizados pelo sistema, veja o arquivo [dspace-source]/config/registries/dublin-core.xml.

Observe também que o dublin-core-types.xml só é usado durante o processo de construção para preencher o Registro de metadados no banco de dados, que é a versão "ao vivo". As alterações que você faz pela interface do usuário só serão refletidas no registro de banco de dados, e não o arquivo XML (e vice-versa.)

Formato do Registro Bitstream

Esta lista de bitstream fornece informações sobre fluxos de bits conhecidos e seu nível de suporte. Você pode editar ou adicionar novos formatos bitstream com esta ferramenta. Mas, por favor, observe o que segue:

  • Você só pode editar um formato bitstream de cada vez. Faça uma fila de edições, e clique no botão 'update'. Só edições feitas para essa seção serão salvas.
  • "Extensões" são extensões de arquivos que são usadas ​​para identificar automaticamente o formato dos arquivos submetidos. Você pode inserir várias extensões para cada formato, separadas por vírgula.
  • O "nome" de cada formato deve ser único, embora o tipo MIME não precisa ser. Por exemplo, você pode ter entradas separadas para formatos de registros em diferentes versões do Microsoft Word, mesmo que o tipo MIME seja o mesmo para cada um deles.
  • Formatos "internos" são escondidos do usuário, e são utilizados para fins administrativos. Por exemplo, as licenças que os depositantes submetem são armazenadas como bitstream de formato "licença", e estes não são exibidos para usuários finais.
  • Se você excluir um formato bitstream, qualquer bitstream para aquele formato no sistema será convertido para formato 'Desconhecido'.
  • Não exclua os formatos 'Unknown' ou 'License'!

Observe também que o config/registries/bitstream-formats.xml só é usado durante o processo de construção do banco de dados, que é a versão "ao vivo". As alterações que você faz neste UI só serão refletidas no registro de banco de dados, e não o arquivo XML (e vice-versa.)

Autorização

Esta seção é usada para definir as políticas específicas de autorização das comunidades, coleções e itens. Para que os usuários possam executar uma ação em um objeto, eles devem ter permissão; o DSpace adota política "negar por padrão". Permissões não são comutáveis; por exemplo, se um e-usuário tem permissão de LER um item, ele não necessariamente tem a permissão sobre os pacotes e bitstreams daquele item.

Gerenciar Políticas de uma Coleção

Utilize esta ferramenta para autorizar os grupos ligados à coleção para executarem suas funções.

  • Clique em Gerenciar políticas de uma Coleção;
  • Selecione o nome da coleção para o qual você deseja definir as autorizações e clique em Editar diretivas;
  • Clique em Adicionar Novo;
  • Selecione o grupo para o qual você deseja definir as autorizações, em seguida, selecione uma ação para esse grupo a partir do menu drop-down.
  • Clique em Salvar Política;
  • Repita até que todas as políticas estejam armazenadas.

Nota:

  • Os itens recém submetidos aceitos em uma coleção herdam as políticas de autorização DEFAULT_ITEM_READ e DEFAULT_BITSTREAM_READ associadas à coleção, que se tornam políticas READ para o item e seus bitstreams. No entanto, se você mudar as políticas padrão de uma coleção após itens terem sido aceitos, as políticas existentes para esses itens não serão alteradas automaticamente. Você terá que alterar as permissões para esses itens usando a ferramenta Advanced Policy Admin Tool para torná-las acessíveis.

  • Todas as coleções devem ter uma política ADD para um grupo de depositantes, ou então ninguém vai ser capaz de enviar itens para a coleção.

Gerenciar Políticas de uma Comunidade

Esta ferramenta permite que você edite as políticas de uma comunidade da mesma forma que as coleções são editadas, conforme descrito acima.

Atualmente, como a administração é feita de forma centralizada, esta ferramenta não tem muito uso - geralmente você só vai adicionar a permissão READ para o grupo Anonymous apenas depois de criar uma comunidade, e deixar desta forma. Esta permissão é aplicada a logomarca da comunidade, se houver uma, o que permite que as pessoas vejam o logotipo quando eles forem para a página inicial da comunidade.

Políticas de uma comunidade são definidas como um padrão Anonymous READ.

Gerenciando políticas de um item

Esta ferramenta permite que você edite as políticas para itens individuais. Quando você quiser que um item tenha uma política diferente do restante dos itens de uma coleção, você pode usar o editor de política de item para personalizar as políticas. No entanto, no longo prazo uma boa prática de gestão de uma coleção é que todos os itens compartilhem as mesmas autorizações.

Ferramenta de Política de Administração Avançada (Advanced Policy Admin Tool)

Utilize esta ferramenta para definir e limpar políticas para itens ou bitstreams para toda a coleção. Tenha cuidado ao usar a botão Clear Policies.

Selecione da lista top a coleção e da lista inferior o grupo que pretende dar ou retirar a permissão. Em seguida, selecione o tipo de objeto (item ou bitstream) para o qual você deseja editar permissões. Em seguida, selecione a ação a partir da caixa drop-down.

Por exemplo, digamos que você queira dar a todos os membros de um grupo de “Usuários privilegiados” um acesso Read a todos os bitstreams de uma coleção restrita. Você iria selecionar 'Usuários privilegiados ' da lista top, 'Bitstream' da caixa drop-down 'tipo de conteúdo', 'usuários privilegiados' na lista 'Grupo', 'Leitura' da caixa drop-down 'Ação', e clicar em 'Adicionar políticas'.

Editar Notícias

Você pode usar esta ferramenta para editar o texto ("notícias") em dois lugares na home page DSpace: a top box da moldura central e na barra lateral direita.

Depois de clicar em "Editar Notícias" no menu do Administrador, clique no botão Editar ao lado da notícia que você deseja editar. Uma caixa de texto será exibida com as notícias atuais, que podem ser eliminadas ou modificadas, digitando diretamente na caixa. Você pode usar tags HTML para formatar o texto, mas note que o html não será validado aqui.

 

Editar Licença Padrão

Você pode usar esta ferramenta para editar a licença padrão do DSpace.

A licença padrão é usada quando nenhuma licença específica de coleção está definida.

Observe que alterar a licença padrão não tem efeito sobre os itens já publicados.

Algumas variáveis ​​de substituição estão disponíveis para configurar uma submissão de licença contextual:

  • %1$s será traduzido para o "primeiro nome do depositante"
  • %2$s será traduzido para o "último sobrenome do depositante"
  • %3$s será traduzido no "email do depositante"
  • %4$s será traduzido para a data atual
  • %5$s será traduzido na informação da coleção
  • %6$s será traduzido na informação do item
  • %7$s será traduzido na informação do usuário

Se você precisa incorporar no texto da licença o símbolo %, você pode evitar a inclusão de um segundo %, escrevendo %%. Recomenda-se usar uma submissão fictícia para verificar a mudança da licença: verifique a licença resultante na etapa Aceitar / Rejeitar licença.
